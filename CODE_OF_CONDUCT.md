# Code of Conduct

The AIMS project adopts the **Contributor Covenant, version 2.1** as its Code of Conduct, in full and without modification. This document records our adoption and explains the few project-specific operational details that any participant needs to know.

## The Covenant

The Contributor Covenant 2.1 is a widely-used Code of Conduct for open-source communities. Adopting it gives newcomers a familiar, well-tested set of expectations rather than a project-specific document they have to learn from scratch.

The full canonical text is published at:

- **Covenant text (HTML):** https://www.contributor-covenant.org/version/2/1/code_of_conduct/
- **Covenant text (Markdown source):** https://www.contributor-covenant.org/version/2/1/code_of_conduct/code_of_conduct.md
- **Translations:** https://www.contributor-covenant.org/translations/

The full text covers:

- A pledge to make participation a positive and constructive experience for everyone
- Examples of behavior that contributes to a positive community
- Examples of behavior that is not acceptable
- Responsibilities of project maintainers
- The scope of where the Code applies (project spaces, public spaces representing the community)
- Enforcement guidelines (a four-tier ladder: correction, warning, temporary suspension, permanent removal)
- Attribution to the Contributor Covenant as the source

A copy of the full canonical text will be vendored into this repository at `CODE_OF_CONDUCT-FULL.md` at first release so the canonical text is always available offline alongside the spec.

## Project-specific operational details

These are the AIMS-project-specific items that supplement the canonical Covenant text.

### Scope

The Code of Conduct applies to all spaces governed by the AIMS project, including:

- This repository (issues, pull requests, discussions, RFCs, code review)
- AIMS-project mailing lists, chat channels, and forums (when stood up)
- AIMS-related events, workshops, and presentations where participants represent the project
- Direct or indirect public references to participation in the AIMS community

It applies equally to maintainers, contributors, reviewers, and casual readers who choose to engage.

### Reporting

To report a Code of Conduct concern, contact the maintainer team at:

- **`aims@xtendedsystems.com`** (the same address used for general AIMS project contact; flag the message subject as "CoC report" to route it appropriately).

Reports are read by the maintainer team. Reporters may request that their identity remain confidential within what's reasonable for the response. The team aims to acknowledge receipt within 72 hours and to provide a substantive response (action taken, action declined with explanation, or status update if the situation is still being assessed) within two weeks.

If a report concerns a maintainer, please send it to a different maintainer and request that the conflicted party be excluded from the response. If all maintainers are conflicted, please escalate to a trusted external party of the reporter's choosing — the project will honor a referral to a recognized Code-of-Conduct enforcement service if one is engaged.

### Enforcement responsibility

Maintainers are responsible for clarifying and enforcing the Code of Conduct in good faith. They may take any action they deem appropriate, up to and including temporary or permanent removal from project spaces, in response to behavior they consider inconsistent with the Covenant.

Decisions are made by maintainer consensus where possible. When consensus is not possible the project lead has the final call, with the rationale recorded privately for accountability and shared with the affected party.

### Appeals

A participant who disagrees with an enforcement decision may request reconsideration by replying to the original notice with new information or context. The maintainer team will reconsider in good faith and respond within two weeks. Appeals are not a forum to relitigate the underlying behavior; they are an opportunity to surface information the original decision did not consider.

### Updates to this document

When the Contributor Covenant publishes a new version, the AIMS project will evaluate whether to adopt it. Adoption decisions are recorded in `governance/decisions/` and announced in the next release notes.
